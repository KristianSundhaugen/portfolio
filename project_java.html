Java Project

An NTNU Gjøvik School Project

Developing an client/server version of the board game Ludo

My first encounter with Java

Summary

  • The Project Assignment

    • The assignment was to make an client/server version of the board game Ludo. Where users could start games between friends or random users. There would also be a chat function between users. The application needed to have permanent storage of user data(games, userinfo, passwords, ets...). Log all chat activity between users. The possibilities for users to request random games. Play the game of course. Getting automatically logged in when starting client. Possibility to change the profile info. Be part of multiple chatrooms. And finally have users games logged, so if they leave during, they can rejoin later. Making it so an user can partake in multiple games at once.

  • Our work

    • So we developed a game with the original ludo rules. Making it so up to four players could join a game(minimum 2).A Any user can at any point join a optional amount of games, but will be kicked if not partaking in a certain time. The players could chat between themselves, our join other chatrooms outside the specific game.

    Our server keeps order over active games and channels with which users that are connected to these. And if all users leave an game or channel it deletes itself from server.

    A game can be started in two ways.

    • A player would challenge another player. Sending an message to them. Then adding which players that he want's to partake. The player receives a challenge, and has a certain time to respond to this. The game starts either when all players has responded, or the time for accepting has run out.
    • The other way a user can join a game is to choose to join an random game. Players that choose this will be added to a queue with up to four players. So as soon there is four players in the queue the game will start with those four players.

    A game continues until there is a winner, or the game has been at a standstill for too long. A game is won when all pieces are in the goal area. We made it so that there is statistics over which user has won most games and which user has partaken in the most games.

    The user interface is built with javaFX and the application is made to have I18N, so the language used for your desktop is what the ingame text will be set at.

  • What I learnt

    • This was my first introduction to working with Java on a big scale. I have worked with backend before, through C++ and such my first two years, but this was definitely an application on a much bigger scale than I was used to. The information handling between client/server was a hugh steppingstone. Took a hell of a lot of time to understand, but in the end got what we wanted. Other than that I found I liked to work with the javaFx, but I think that jus might be because I have a love for any Front-End work. The feeling of seeing what I am developing makes it som much more rewarding.
